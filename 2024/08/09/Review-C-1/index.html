从新的角度理解声明语句

从一个例子开始：

请问，p1、p2分别是什么类型？可能会有人认为p1和p2都是是指针类型，但其实p1是指向int的指针类型，即int *类型；p2是int类型。

这是一个C语言初学者常犯的错误，如果我们认为C语言的声明语句就是【类型名】+【变量名】，那么对于上面的例子我们就会自然的认为p1、p2都是int* 类型。其实这样理解C语言中的声明语句是不准确的。更准确的说法应当为：

  • 【朴素类型名】 +【表达式】

其中朴素类型名包括C语言中的基本类型（如：int、char、unsigned long等）、结构体类型、共用体类型以及由typedef定义的类型。但是不包括由上述类型派生出的指针类型。

其后的表达式为包含一个合法标识符的表达式，表达式中只允许下述操作：解引用*、函数调用(...)、数组取值[..]。

二者结合起来，声明语句的含意为对标识符执行表达式中的操作后，得到的值的类型为【朴素类型名】代表的类型。

那么，有了新的理解，我们也可以对指针、函数、数组、函数指针等的声明语句进行重新解释。例如

  1. 指针声明

    • *p1是int类型。对p1解引用得到int类型，所以p1是指向int类型的指针
    • **p2是int类型。对p2两次解引用得到int类型，所以p2是指向int类型的二重指针
    • ***p3是int类型。对p3三次解引用得到int类型，所以p3是指向int类型的三重指针

  2. 函数声明

    • add(int a, int b)的类型是int。对add进行函数调用得到的值是int，所以add是一个返回int类型的函数，且接收参数列表为(int a, int b)。

  3. 函数指针与返回指针的函数

    • 第一行：*f1(int)是int类型，根据C语言的运算符优先级，函数调用()优先于解引用*，因此f1首先可以进行函数调用，调用结果进行解引用得到int类型，因此f1是一个返回值为int *类型的函数。
    • 第二行：(*f2)(int)是int类型，由于括号内优先计算，因此f2首先可以解引用，然后进行函数调用，得到int类型。因此，f2是一个函数指针，指向一个返回值为int的函数。

  4. 数组声明

    • str[100]是char类型，对str进行下标运算得到char，因此str是char类型数组
